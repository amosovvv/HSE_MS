Отчёт о выполнении домашнего задания «Big Data и ML»

Цель работы
Спроектировать и реализовать аналитический pipeline:
PostgreSQL => ETL => MinIO => обработка => Jupyter => витрины => Superset

1. Развёрнутая инфраструктура (Docker Compose)
PostgreSQL 15 – исходное хранилище данных.

MinIO – объектное хранилище (S3-совместимое) для хранения очищенных и агрегированных данных в формате Parquet.

Jupyter (datascience-notebook) – среда для ETL, очистки, feature engineering и построения ML-моделей.

Superset – BI-инструмент для визуализации витрин.

Все сервисы объединены общей сетью, настроены healthcheck и автоматическое создание bucket в MinIO.

2. ETL: выгрузка данных в MinIO
Из PostgreSQL в Jupyter через pandas загружены таблицы:
wells, production, well_telemetry, well_targets, pumps, pump_sensors, pump_failures, deliveries, drivers, vehicles, oil_stations.

Данные преобразованы (очистка NULL, фильтрация выбросов по z-score) и сохранены в MinIO в формате Parquet с партиционированием по дате (year/month/day) для таблиц:
production
well_telemetry
pump_sensors
deliveries

Справочные таблицы сохранены как единые файлы.

3. Очистка и подготовка данных (Jupyter)
Обработка NULL – заполнение медианными значениями по группам (скважинам).

Фильтрация выбросов – удаление записей с |z-score| > 3 по дебиту.

Создание признаков:
коэффициент простоя = downtime_hours / 24
среднее давление, температура (по дням, скважинам)
время работы насосов = 24 – downtime_hours

Агрегации:
общая добыча по дням
KPI по скважинам (средний дебит, % простоя, avg давление/температура)

Сохранены витрины:
daily_production.parquet
well_kpi.parquet
ml_dataset.parquet (признаки + целевая переменная)

4. Выполнение практических заданий
Задание 1. Аналитика добычи
Построены витрины в PostgreSQL: daily_production, well_kpi, production_clean.

В Superset созданы дашборды:
Line chart – общая добыча по дням.
Bar chart – топ скважин по среднему дебиту.
Heatmap – давление vs дебит (с цветовым кодированием температуры).
Выявлены лучшие/худшие скважины.

Задание 2. Прогноз дебита (ML)
Датасет: из production и well_targets (признаки: давление, температура, энергопотребление, время работы насосов).

Обучены Linear Regression и Random Forest.
Метрики (Random Forest): MAE ≈ 1.2 т, RMSE ≈ 1.8 т.
Прогноз сохранён в таблицу forecast_results.

В Superset визуализированы:
Scatter plot Actual vs Predicted
Временной ряд ошибки модели.

Задание 3. Аномалии и отказ оборудования
На данных pump_sensors и pump_failures:
Найдены аномалии через z-score (вибрация, температура, ток, обороты).
Обучена логистическая регрессия для оценки вероятности отказа (ROC-AUC = 0.85).
Сохранены таблицы: pump_anomalies, pump_risk_latest.

В Superset:
График аномалий по времени.
Рост вибрации перед отказом.
Risk score по насосам.

Задание 4. Логистика и поставки
Проанализированы факторы задержек: погода (макс задержка при снеге), расстояние (корреляция 0.45), водители (KPI).

Рассчитана стоимость на км (cost_per_km), найдены оптимальные маршруты (минимальные cost_per_km + задержка).

В Superset построены:
Delay vs Weather – bar chart.
Cost vs Distance – scatter plot с группировкой по погоде.
Таблица KPI водителей (средняя задержка, объём, стоимость на км).

5. Результаты и выводы
Полностью реализован pipeline: PostgreSQL => MinIO => Jupyter => витрины => Superset (через PostgreSQL).
Данные очищены, обогащены признаками, подготовлены для аналитики и ML.

Построены 4 интерактивных дашборда в Superset, охватывающих:
мониторинг добычи
прогнозирование дебита
обнаружение аномалий и риск отказов оборудования
оптимизацию логистики
ML-модели позволяют с приемлемой точностью предсказывать суточную добычу и оценивать риск отказа насосов.
